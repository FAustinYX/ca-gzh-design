# 非技术同事的纯文本结构化规则

目标：员工只粘贴普通文本，技能在内部生成 Markdown，不要求员工学习或修改 Markdown。

## 输入清理

1. 删除重复空行、微信复制产生的多余空格和孤立不可见字符。
2. 保留链接、数字、费用、日期、学校名、课程名和图片占位。
3. 中文正文统一全角标点；URL、邮箱、课程代码、英文缩写保持原样。
4. 不改写事实，不自动更新政策，不补造来源。

## 结构推断

- 文首最能概括全文的短行 → `#` 标题。
- `一、`、`1.`、`01`、`Step 1`、`Day 1` 或前后有空行的短行 → `##` 章节。
- 章节内部的步骤、问题、学校名或日期段 → `###` 小节。
- `-`、`·`、`✔`、`❌`、连续短行 → 列表。
- “注意、提醒、必须、不要、截止、仅限” → 提示或警告块候选。
- 连续问答句 → FAQ。
- 末尾咨询、报名、进群、扫码 → CTA；合并成一个区域。

全文没有标题时，根据语义切成 3–6 个章节并自拟不超过 12 字的标题。标题使用结论或动作，例如“申请前准备”“真正的风险在哪”，不要使用“第一部分”。

## 四类内容的额外识别

- 教程：把材料与资格放在步骤前，步骤保持原始顺序。
- 活动：把时间、地点、Day、地址提取为独立结构。
- 风险：把现象、原因、后果、误区、解决方案分开。
- 快讯：先按学校/机构分组，再提取变化、影响和行动。

## 结构确认

默认只向用户展示：文章类型、章节数量、章节名称、图片数量和自拟标题。不要展示整份 Markdown。

用户说“一键”“直接排”“不用问”时跳过确认，交付时附结构说明。
