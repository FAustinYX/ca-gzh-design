# CA 公众号排版

将普通原稿自动结构化并输出公司统一风格的微信公众号 HTML。把 Markdown 仅作为内部中间格式，除非用户主动要求，否则不要让用户编辑或理解 Markdown。

## 必读资源

按任务读取：

1. 所有任务先读 `references/company-brand-guidelines.md` 与 `references/company-template-routing.md`。
2. 输入不是 Markdown 时读 `references/plaintext-to-markdown.md`；Word 同时使用 `scripts/extract_docx.py`。
3. 选定模板后读对应主题：
   - 公司蓝：`references/theme-company-blue.md`
   - 公司红：`references/theme-company-red.md`
4. 出现代码、图片、GIF、素材占位、通用小标签或文末咨询二维码时读 `references/common-components.md`。
5. 只有用户明确要求公司体系以外的风格时，才读 `references/theme-index.md` 选择扩展主题。

具体 HTML 必须从主题组件库与通用组件库取，不凭记忆重写组件。

## 默认交互

- 用户粘贴普通文章并说“排版”：直接识别结构、自动选模板、生成结果，不要求用户先转 Markdown。
- 用户说“直接排”“一键排版”“不用问”：全自动完成，不进行中途确认。
- 用户未说全自动，但结构存在重大歧义：只展示章节清单让用户确认一次，不展示 Markdown 源码。
- 用户指定模板时直接使用；未指定时按模板路由自动选择，并在交付时说明选择结果。
- 不把技术名词、脚本路径或 Markdown 教程暴露给普通使用者。

## 工作流

### 1. 归一化输入

把纯文本、Word、PDF 或网页内容转换为内部 Markdown 草稿：

- 保留原文事实、数字、链接、图片和实质内容。
- 识别文章标题、章节、步骤、列表、提示、FAQ、CTA 和署名。
- 允许修正明显错别字、半角中文标点与多余空行。
- 不擅自补造政策、日期、费用、学校要求或申请条件。
- 自拟章节标题时必须贴合原意，且在交付说明中列出。

### 2. 判定内容类型

根据 `company-template-routing.md` 判定一个主类型：

- 教程攻略
- 活动指南
- 风险洞察
- 院校快讯

内容复合时选主类型，局部使用同主题内的辅助组件；不要跨公司蓝、公司红混用整套组件。

### 3. 套用公司设计语言

严格执行 `company-brand-guidelines.md`：

- 信息、行动、学校与活动使用公司蓝。
- 风险、误区、强警示使用公司红。
- 黄色只用于提醒、福利、限时和关键注意事项。
- 正文保持深灰、白底、15px 左右，避免整页高饱和。
- 每个正文段落最多标记 1–2 个关键词；没有重点可以不标。
- 最强视觉锚点全文不超过 5 处。

### 4. 按配方装配 HTML

从对应主题文件的“文章类型 → 组件组合配方”与“完整文章模板骨架”选组件：

- 教程：问题引入 → 准备清单 → 编号步骤 → 注意事项 → FAQ → CTA。
- 活动：情绪封面 → 活动价值 → 时间地点 → 日程/路线 → 福利提示 → 行动 CTA。
- 风险：反常识引言 → 风险机制 → 典型误区 → 数据/门槛 → 解决方案 → 服务 CTA。
- 快讯：本周摘要 → 院校分区 → 变更要点 → 影响对象 → 截止/行动提醒 → 咨询 CTA。

原文已有图片时保留原顺序；只有原稿明确提供说明时才生成图注。不得保留虚假图片 URL 或素材占位。

默认在唯一咨询 CTA 中加入公司二维码，使用 `common-components.md` 的 company-qr-cta：

- 教程、风险、快讯：放在结论或服务说明之后，互动三连、END 或品牌尾图之前。
- 活动：放在福利、地址和预约说明之后；若正文已有群聊入口，将其合并为同一个二维码 CTA。
- 全文只出现一次公司二维码；用户明确要求不加时才省略。

### 5. 微信兼容与校验

产物必须是纯 `<section>…</section>` 正文片段：

- 不得包含 `html`、`head`、`body`、`style`、`script`、`div`。
- 样式全部内联。
- 所有文字节点用 `<span leaf="">` 包裹。
- 禁止 class、id、CSS 变量、外部 CSS、外部字体、grid、float、fixed、absolute、sticky。
- 中文正文使用全角标点；代码、URL、英文专名保持原样。

写出 HTML 后先嵌入固定品牌素材，再强制校验：

```bash
<SKILL_ROOT>/scripts/embed_company_assets.py <OUTPUT_HTML>
```

该脚本把 `{{COMPANY_QR_SRC}}` 替换为 Skill 内置二维码的数据 URI，保证本地预览和复制时不依赖临时文件路径。然后运行：

```bash
<SKILL_ROOT>/scripts/validate_gzh_html.py <OUTPUT_HTML>
```

修复到 ERROR 为 0；正文半角标点 WARNING 也修复到 0。然后运行：

```bash
<SKILL_ROOT>/scripts/ensure_utf8_bom.py <OUTPUT_HTML>
<SKILL_ROOT>/scripts/wrap_preview.py <OUTPUT_HTML>
```

先给干净正文添加 UTF-8 BOM，避免 Safari 在正文片段没有 `<head>` 时误判中文编码。预览页必须同时包含 `<meta charset="UTF-8">` 并以 UTF-8 BOM 写出。

### 6. 交付

交付两个文件：

1. `{原文件名}_公司排版_{模板}.html`：干净正文。
2. `{原文件名}_公司排版_{模板}_预览.html`：带“一键复制到公众号”按钮。

用普通语言告知用户：打开预览文件，点击“复制到公众号”，然后粘贴到公众号后台。附上：内容类型、采用模板、自动识别章节、自拟标题、校验结果。

## 内容安全与质量规则

- 政策、签证、学校要求和日期可能变化；只排版用户提供的内容，不把排版过程当作事实核验。
- 若用户要求事实核验，单独标记需要核验的句子并使用权威来源。
- 不因“优化阅读体验”删除限制条件、例外、费用、截止日期或免责声明。
- 营销 CTA 保持克制，一篇文章只保留一个主要 CTA 区域。
- 原文已有 CTA 时合并到唯一 CTA，不重复生成。
- 未提供作者、二维码、联系方式时保留可读的通用结尾，不留下 `{{占位符}}`。
